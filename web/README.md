# thoughtml-web

A live editor + graph playground for ThoughtML v0 — like the Mermaid live editor,
but for reasoning graphs. Type ThoughtML on the left; see the canonical object
graph (spec §13) on the right, updating as you type.

It runs the **real Rust parser** compiled to WebAssembly, so the browser and the
CLI can never disagree about what a document means.

## Architecture

```
CodeMirror 6 editor ──debounce(200ms)──▶ wasm parse(src)
                                              │  { canonical, diagnostics, surface }
        ┌─────────────────────────────────────┼───────────────────────────┐
        ▼                     ▼                ▼                           ▼
   Cytoscape graph     diagnostics panel   editor lint gutter        JSON / AST tabs
   (§13 projection)    (click → jump)      (line markers)
```

**The view is a wasm-free core.** The renderer depends only on the canonical
model types (`model.ts`), never on the parser — so the same code drives the
live playground (`index.html` / `main.ts`, with wasm) and the standalone viewer
(`viewer.html` / `viewer.ts`, model baked in, no wasm).

- `src/model.ts` — the canonical-model **type contract** + pure helpers, zero
  wasm. The seam between the compiler and the view.
- `src/parse.ts` — the wasm adapter (`initParser()` / `parse()` …), importing its
  types from `model.ts`. Only live hosts need it.
- `src/graph.ts` — projects canonical objects to a Cytoscape graph *(core)*.
- `src/detail.ts` — the node-detail panel *(core)*.
- `src/legend.ts` — the legend + lens keys, shared by both shells *(core)*.
- `src/icons.ts` — glyphs *(core)*.
- `src/editor.ts` — CodeMirror 6 with a ThoughtML highlight mode *(playground)*.
- `src/diagnostics.ts` — the diagnostics panel *(playground)*.
- `src/main.ts` — playground wiring: tabs, examples, view toggle, what-if, split.
- `src/viewer.ts` — the standalone read-only viewer (graph + detail + lenses +
  as-of, no editor / examples / what-if / wasm).
- `src/wasm/` — generated by `wasm-pack` from `../crates/thoughtml-wasm` (not edited by hand).

## Graph views

Links and stances are reified as nodes when something else references them, so
`link → link` and `stance → anything` always render correctly.

- **Readable** (default): links/stances that nothing points at collapse into
  labelled edges (`a —causes→ b`, `team —suspects(0.25–0.70)→ X`).
- **Structural**: every object is a node, faithful to the §13 property graph.

## Develop

Requires `rustup` with the `wasm32-unknown-unknown` target and `wasm-pack` on PATH
(`~/.cargo/bin`).

```sh
npm install
npm run wasm          # build the parser to src/wasm/ (re-run after parser changes)
npm run dev           # start the Vite dev server (playground at /, viewer at /viewer.html)
npm run build         # typecheck + production bundle to dist/
npm run build:viewer  # rebuild the single-file viewer template the CLI embeds
```

`npm run wasm` must be run at least once before `dev`/`build`, and again whenever
the Rust parser or bindings change.

`npm run build:viewer` inlines the viewer into one self-contained HTML at
`../crates/thoughtml/assets/viewer.html` — the template the `thoughtml --html`
CLI bakes a document's canonical JSON into. Re-run it after changing any viewer
or core file; a CI guard (`cargo test -p thoughtml --test viewer_freshness`)
fails if the committed template is stale.

resizable split, node inspector, and a **standalone wasm-free viewer** exported
via `thoughtml --html`.
